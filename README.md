# 프롬프트 캐싱 / 컨텍스트 관리

채팅 컨텍스트가 사라지거나 새 채팅으로 옮겨가도, 문서를 읽어 진행 맥락과 기획 의도를 유지하기 위한 문서 구성입니다.

**참고:** 아래 문서들은 프롬프트에서 자동으로 인식되지 않습니다. 필요할 때 해당 문서를 직접 참조(예: `@파일명`)하거나 읽어 주세요.

---

## 1. HANDOFF.md _(프로젝트 루트)_

- **위치:** 프로젝트 루트 (현재는 미생성)
- **역할:** 바이브 코딩 등 작업을 하다가 **진행 중인 상황**을 별도로 저장하는 문서
- **용도:**
  - 컨텍스트 캐시가 없어졌을 때
  - 새 채팅으로 이동했을 때  
    → 이 문서를 읽으면 **전체적인 진행 맥락**을 파악할 수 있도록 함

---

## 2. project-description.md _(프로젝트 루트)_

- **위치:** 프로젝트 루트 (현재는 미생성)
- **역할:** 해당 프로젝트의 **전체 기획서**
- **용도:** 개발하면서 이 문서를 읽고 **프로젝트 기획 의도**를 파악하기 위함

---

## 3. skills/ — 공통 원칙 및 표준

### 3.1 programming-standards.md _(가장 우선)_

- **역할:** **1원칙**으로 지켜야 할 기초 문서
- **범위:** 모든 프로그래밍 언어에 공통
- **성격:** 프로그래밍과 **커뮤니케이션** 관련 의도를 정리한 문서에 가까움

### 3.2 skills/기타

- **역할:** **언어·라이브러리·프레임워크·엔진** 등에 대한 설명 문서
- **예:** 언어별 코딩 표준, 사용 중인 기술 스택 설명 등

_(추가 문서는 이후 작성 예정)_

## 문서 사용 요약

| 문서                              | 용도                                    |
| --------------------------------- | --------------------------------------- |
| `HANDOFF.md`                      | 진행 상황·맥락 복원 (캐시/채팅 전환 시) |
